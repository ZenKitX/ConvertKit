# ConvertKit 性能基准测试

本目录包含 ConvertKit 各个编解码器的性能基准测试。

## 运行基准测试

### 运行所有基准测试

```bash
dart run benchmark/hex_benchmark.dart
dart run benchmark/percent_benchmark.dart
```

### 运行单个基准测试

```bash
# 十六进制编解码基准测试
dart run benchmark/hex_benchmark.dart

# 百分号编解码基准测试
dart run benchmark/percent_benchmark.dart
```

## 基准测试说明

### hex_benchmark.dart

测试十六进制编解码器的性能，包括：

- **编码性能**: 将字节数组转换为十六进制字符串
- **解码性能**: 将十六进制字符串转换为字节数组
- **往返转换**: 编码后再解码的完整流程

测试数据规模：
- 小数据: 100 字节
- 中等数据: 10KB
- 大数据: 1MB

### percent_benchmark.dart

测试百分号编解码器的性能，包括：

- **编码性能**: 将字节数组转换为百分号编码字符串
- **解码性能**: 将百分号编码字符串转换为字节数组
- **往返转换**: 编码后再解码的完整流程

测试数据规模：
- 小数据: ~50 字节（包含特殊字符和中文）
- 中等数据: ~50KB
- 大数据: ~500KB

## 性能指标

每个基准测试输出以下指标：

- **总时间**: 所有迭代的总耗时（毫秒）
- **平均时间**: 单次操作的平均耗时（毫秒）
- **吞吐量**: 每秒处理的数据量（MB/s）

## 优化建议

如果基准测试显示性能问题，可以考虑：

1. 使用流式 API（`Converter.startChunkedConversion`）处理大数据
2. 对于频繁的小数据转换，考虑缓存结果
3. 检查是否有不必要的内存分配
4. 使用 `dart compile exe` 编译为原生代码以获得更好的性能

## 注意事项

- 基准测试结果受系统负载、CPU 频率等因素影响
- 建议在相对空闲的系统上运行以获得稳定结果
- 多次运行取平均值可以获得更准确的性能数据
- JIT 编译器需要预热，首次运行可能较慢
